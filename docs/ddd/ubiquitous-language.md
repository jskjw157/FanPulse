## Identity Context (인증)

| 한글 | 영문 | 정의 | 사용 예시 |
|------|------|------|----------|
| 사용자 | User | 서비스에 가입한 회원. 이메일 또는 OAuth로 가입 가능 | "사용자가 로그인했습니다" |
| 회원가입 | Registration | 새로운 사용자 계정을 생성하는 과정 | "회원가입 시 이메일 인증 필요" |
| 로그인 | Login | 사용자가 서비스에 인증하는 행위 | "로그인에 성공했습니다" |
| 인증 토큰 | Auth Token | 사용자 세션을 유지하기 위한 JWT 토큰 | "인증 토큰이 만료되었습니다" |
| 액세스 토큰 | Access Token | API 호출 시 사용하는 단기 유효 토큰 | "액세스 토큰을 갱신합니다" |
| 리프레시 토큰 | Refresh Token | 액세스 토큰 갱신을 위한 장기 유효 토큰 | "리프레시 토큰으로 재인증" |
| OAuth 계정 | OAuth Account | 소셜 로그인으로 연동된 외부 계정 (Google 등) | "Google OAuth 계정 연동 완료" |
| 사용자 설정 | User Settings | 테마, 언어, 알림 등 개인화 설정 | "사용자 설정을 변경했습니다" |
| 닉네임 | Username | 서비스 내에서 표시되는 사용자 이름 | "닉네임을 입력해주세요" |
| 세션 | Session | 로그인 상태가 유지되는 기간 | "세션이 만료되었습니다" |
| 로그아웃 | Logout | 현재 세션을 종료하는 행위 | "로그아웃 되었습니다" |
