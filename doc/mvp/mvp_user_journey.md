# 🧭 FanPulse MVP User Journey

## 1) 목적
MVP 사용자 경험의 핵심 흐름을 정의하고, 정상/오류/빈 상태를 명확히 한다.

---

## 2) 핵심 플로우 (Happy Path)
1. 앱 실행 → 로그인 화면 진입
2. 이메일/비밀번호 또는 Google 로그인 성공
3. 홈(H001)에서 라이브/뉴스 카드 탐색
4. 라이브 카드 탭 → 라이브 상세(H019)에서 임베드 시청
5. 뒤로가기 → 홈 복귀
6. 뉴스 카드 탭 → 뉴스 상세(H011) 조회
7. 검색(H018)에서 Live/News 통합 검색
8. 마이페이지(H016) → 설정(H010) → 로그아웃

---

## 3) 서브 플로우
### 3.1 회원가입
- 로그인 화면에서 회원가입 탭 전환 → 이메일/비밀번호 입력 → 가입 완료 → 로그인

---

## 4) 오류/빈 상태
### 4.1 네트워크 오류
- API 실패 시 공통 에러 화면(H024) 노출
- 재시도 버튼 제공

### 4.2 인증 오류
- 토큰 만료/부재 시 로그인 화면으로 리다이렉트

### 4.3 빈 상태
- 라이브 목록 0건: “현재 진행 중인 라이브가 없습니다”
- 뉴스 목록 0건: “표시할 뉴스가 없습니다”
- 검색 결과 0건: “검색 결과가 없습니다”

---

## 5) 화면 간 연결 요약
- H002 → H001 → H006 → H019
- H001 → H011
- H001 → H018
- H001 → H016 → H010 → H002
