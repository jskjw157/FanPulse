# 📄 FanPulse MVP PRD

## 1) 목적/배경
FanPulse는 K-POP 팬을 위한 플랫폼으로, MVP에서는 로그인 후 **라이브/뉴스를 안정적으로 탐색·시청**할 수 있게 한다.
운영 리스크를 줄이기 위해 데이터는 **seed 기반 적재**로 시작한다.

---

## 2) 목표 (Goals)
- 로그인 후 홈에서 라이브/뉴스를 탐색하고 상세까지 도달한다.
- iOS/Android/반응형 Web에서 동일 플로우로 동작한다.
- 크래시/치명 버그 0, 로딩/에러/빈 상태 처리 완료.

## 3) 비목표 (Non-goals)
- 커뮤니티, 투표, 광고&리워드, 멤버십, 고객센터 상세 기능.
- 실시간 채팅/하트/도네이션 등 라이브 인터랙션.
- 자동 크롤링/추천/AI 운영 적용.

---

## 4) 대상 사용자/시나리오
### 핵심 사용자
- K-POP 팬 (국내/해외), 라이브/뉴스 소비 중심

### 핵심 시나리오 (Happy Path)
1. 로그인/회원가입
2. 홈에서 라이브/뉴스 카드 탐색
3. 라이브 상세에서 임베드 시청
4. 뉴스 상세 조회
5. 검색으로 라이브/뉴스 탐색

---

## 5) 범위 (Scope)
### 포함 (In)
- Auth: 이메일/비밀번호 + Google 로그인
- Home: 라이브/뉴스 요약 카드
- Live: 목록/상세 (YouTube 임베드)
- News: 목록/상세
- Search: Live/News 통합 검색
- My/Settings: 프로필 요약, 로그아웃, 기본 설정
- Error: 공통 에러 안내

### 제외 (Out)
- Community, Voting, Rewards, Membership, Concert, Support 상세 기능
- 실시간 상호작용(채팅/하트)
- AI 요약/필터링 운영 적용

---

## 6) 기능 요구사항 (요약)
- Auth: 회원가입/로그인/로그아웃, 토큰 기반 인증
- Live: 상태별 목록, 상세 메타데이터, 임베드 재생
- News: 요약 목록, 상세 본문
- Search: Live/News 통합 검색
- UI 상태: 로딩/에러/빈 상태

---

## 7) 데이터/운영
- seed 기반 적재로 `artists`, `streaming_events`, `crawled_news`를 유지
- MVP 크롤링 범위는 Live/News만 포함(차트/콘서트/광고는 Next)
- 운영 플로우: artists → live/news 순서로 upsert

---

## 8) 성공 지표 (예시)
- 로그인 성공률 ≥ 95%
- 홈→라이브 상세 진입률 ≥ 40%
- 앱 크래시율 ≤ 1%
- API 5xx 비율 ≤ 1%

---

## 9) 제약/의사결정
- 라이브는 YouTube 임베드 URL만 지원
- MVP는 seed 적재 중심, 자동 크롤링은 Next
- iOS/Android/Responsive Web 동시 지원

---

## 10) 리스크/대응
- 외부 라이브 링크 변경 → seed 정기 점검
- 데이터 부족 → seed 운영 프로세스 고정
