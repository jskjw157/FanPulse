# 🚀 FanPulse MVP 기획서 (4주)

## 1. MVP 목표

**핵심 가치(4주 MVP)**:
- 로그인(이메일/비번 + Google) 후 **홈에서 라이브/뉴스를 탐색**하고
- **라이브 시청** 및 **뉴스 상세 보기**까지 가능하다.

**성공 기준(예시)**:
- iOS/Android/반응형 Web에서 동일 플로우로 시청 가능
- 크래시/치명 버그 0, 주요 화면 로딩/에러/빈상태 처리 완료

---

## 2. MVP 범위 (Scope)

### 포함(In)
- **Auth**: 로그인(H002), 회원가입(H002-1)
  - 이메일/비밀번호 회원가입 + 로그인
  - 소셜 로그인: **Google 1종**
- **Home**: 메인(H001)
  - Live Now/Upcoming, 최신 뉴스 섹션(요약 카드) 중심
- **Live**: 라이브 & 이벤트(H006), 라이브 상세(H019)
  - 외부 플랫폼(YouTube Live) **임베드 시청**
  - 채팅/하트/투표/리워드 등은 제외
  - **임베드 방식**: Web은 iframe, 앱은 WebView 사용 (상세: `mvp_화면_정의서.md` H019 참조)
- **News**: 뉴스 목록/상세(H001/H011)
  - MVP는 seed 기반 적재(큐레이션)로 시작 + **Week 4에 Google News RSS(Stretch)로 운영 부담 완화**
- **Search**: 검색(H018)
  - MVP 범위: Live/News 통합 검색
- **Local Only**: 저장한 게시물, 최근 검색어
  - 계정 동기화는 다음(Next)
- **My/Settings**: 마이페이지(H016), 설정(H010)
  - 최소 기능: 프로필 요약 + 로그아웃 + 앱 정보/테마(선택)
- **Error**: 에러 페이지(H024) (공통 에러/권한/네트워크 실패 안내)

### 제외(Out / Next)
- 커뮤니티(H003~), 투표(H004), 광고&리워드(H008), 멤버십(H009)
- 고객센터(H023~) 세부 기능
- 라이브 채팅/하트/도네이션 등 실시간 인터랙션
- AI(뉴스 요약/댓글 필터링) 운영 적용 (PoC/설계만 가능)

## 3. DDD 적용 방식(현실형)

### Bounded Context (MVP 최소 2개)
1) **Identity**: 사용자/인증(회원가입, 로그인, 토큰, Google OAuth)
2) **Live**: 스트리밍 이벤트(목록/상세, 상태, 임베드 URL)
3) **Content**: 뉴스(목록/상세) + 검색(읽기 모델 중심)

> 권장: 백엔드는 **모듈러 모놀리스 + 헥사고날(Port/Adapter)** 로 시작(마이크로서비스 분리 금지).

### 계약 우선(Contract-first)
- OpenAPI로 API 계약(요청/응답/에러)을 먼저 고정
- iOS/Android/Web은 계약 기반으로 병렬 개발

---

## 4. 4주 실행 계획(주간 데모)

### Week 1 — 스코프 락 + 뼈대
- (PM/기획) 포함/제외 확정, 수용기준/테스트케이스 작성
- (백엔드: 정지원) DDD 컨텍스트/모듈 구조 결정, OpenAPI v1 작성
- (iOS/Android/Web) 화면 스켈레톤 + 더미데이터로 플로우 데모

**데모**: 로그인/회원가입 화면 → 홈(H001) → 라이브 상세(더미 임베드) → 뉴스 상세(더미)

### Week 2 — Auth E2E
- (백엔드) 회원가입/로그인/토큰/인증 미들웨어 구현
- (클라이언트) 토큰 저장/세션 유지/가드 라우팅/로그아웃

**데모**: 실제 백엔드 붙은 이메일/비번 + Google 로그인 + 로그아웃 + 보호 라우팅

### Week 3 — Live/News E2E + Seed 적재
- (백엔드) 라이브 목록/상세 API + 뉴스 목록/상세 API
- (적재: 정지원, 나영민) seed 기반 upsert로 `streaming_events` + `crawled_news` 채움
- (클라이언트) 실제 데이터로 Home/Live/News 연동

**데모**: 실데이터로 홈→라이브 재생 + 홈→뉴스 상세

### Week 4 — QA/릴리즈
- 공통: 로깅/에러처리/성능, 회귀 테스트, 릴리즈 체크리스트
- (Stretch) Google News RSS → `crawled_news` 자동 upsert(일 1회) + 최신 N개 유지
- 배포: Web preview + iOS TestFlight + Android 내부테스트
